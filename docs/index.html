How it works

Four nodes · One retry loop · One cited report

Step 01 · Planner
Charting the route
Decomposes your topic into 3–7 focused, non-overlapping sub-questions using the configured LLM.
Step 02 · Searcher
Surveying the terrain
Runs DuckDuckGo searches for each sub-question. Deduplicates results via FAISS cosine similarity before storing them.
Step 03 · Evaluator
Verifying landmarks
Scores every source 1–5 for relevance. Checks that each sub-question has enough high-quality sources before proceeding.
Step 04 · Synthesizer
Drawing the map
Generates a structured Markdown report with inline [N] citations and an auto-appended References section.
Self-correcting loop — if the Evaluator finds coverage thin, it routes back to the Searcher for another survey cycle (up to max_retries times) before falling through to synthesis.

Tech stack

Local-first · No vendor lock-in

Layer Tool Why
Orchestration LangGraph Production-grade; native cycles and conditional routing
LLM Ollama Fully local — zero API cost, complete privacy
Web search ddgs No API key required; anyone can clone and run
Deduplication FAISS + MiniLM Semantic similarity prevents redundant sources across sub-queries
UI Gradio Streaming progress, zero frontend code

Quick start

Prerequisites: Python 3.11+ · Ollama installed

# Clone and install
$ git clone https://github.com/surabhi/cartograph.git && cd cartograph
$ uv venv && source .venv/bin/activate
$ uv sync

# Pull a model and start
$ ollama pull llama3.1
$ python app.py

# Open http://localhost:7860
